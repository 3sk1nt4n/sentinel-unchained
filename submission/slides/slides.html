Unchained

GPT‑5.6 chooses where to look; deterministic code controls what may run and verifies exactly what was executed and cited.

A bounded autonomous DFIR investigator, built with Codex and GPT‑5.6.  ·  Judges can run it in about a minute, for $0, with no API key.

The problem

A transcript is not an answer.

  • Agents now touch real evidence - and DFIR conclusions go in front of regulators and opposing counsel.
  • A chat log cannot prove which tools actually ran, what output was retained, or what exact text a claim cites.
  • If the AI’s work cannot be checked independently, it is not evidence. It is a story.
The idea

Split strategy from authority.

GPT‑5.6 decides

  • Where to look: an opening book of up to twelve typed forensic tools
  • One typed action per adaptive turn, against a visible ledger
  • How to interpret output; which findings to propose

Deterministic code owns

  • What may run: typed, route‑eligible, read‑only, all‑or‑none
  • SHA‑256 custody and hard caps that fire before dispatch
  • Exact UTF‑8 citation spans, the report, the sealed bundle, offline verification
Architecture

One bounded lifecycle, receipts at every step.

deterministic code authority GPT‑5.6 judgment inside a bounded protocol 4 fixed GPT‑5.6 requests + one per adaptive action - never an unbounded loop

Honest by design

Fail closed. Label everything.

  • Caps fire before dispatch; a capped run ends PARTIAL - never relabeled COMPLETE.
  • Receipts prove what ran and what was cited - not forensic truth. A human owns interpretation.
  • One case, one OS route; the same‑evidence benchmark was deliberately cut - so no unmeasured claims, anywhere.
